# MediaDive NCBITaxon Dependency on BacDive

MediaDive does **not** do its own NCBITaxon lookups. It relies on BacDive's pre-resolved mappings.

---

## MediaDive Approach

1. **Loads BacDive's output** (`bacdive.tsv`) to get strain→NCBITaxon mappings
2. For medium-strain associations:
   - Looks up BacDive ID in the precomputed `bacdive_df`
   - Uses whatever NCBITaxon ID BacDive determined

## MediaDive Cases

| Case | Condition | Result |
|------|-----------|--------|
| A | Strain in BacDive WITH NCBITaxon | Uses that NCBITaxon ID |
| B | Strain in BacDive WITHOUT NCBITaxon | Uses `kgmicrobe.strain:bacdive_XXXX` |
| C | Strain NOT in BacDive | Uses `kgmicrobe.strain:bacdive_XXXX` |

## Key Difference from BacDive

- **BacDive**: Does NCBITaxon lookup (API + fallback search)
- **MediaDive**: References BacDive's results, no independent lookup

## Benefit

The fallback search added to BacDive **indirectly helps MediaDive** - if BacDive's fallback search finds an NCBITaxon for a Case 3 record, MediaDive will pick that up from `bacdive.tsv`.

---

# `bacdive.tsv` Explained

**Location:** `kg_microbe/transform_utils/bacdive/tmp/bacdive.tsv`

**What it is:** An intermediate TSV file generated by the BacDive transform as a side effect - a denormalized view of BacDive data.

**Generated by:** `bacdive.py` line 818-1230

## Columns

| Column | Description |
|--------|-------------|
| `bacdive_id` | BacDive record ID (e.g., "bacdive:123") |
| `dsm_number` | DSM culture collection number |
| `culture_collection_number` | External links culture number |
| `ncbitaxon_id` | **Resolved** NCBITaxon ID (from API or fallback) |
| `ncbitaxon_description` | Description from BacDive |
| `keywords` | Keywords |
| `medium_id` | Medium ID |
| `medium_label` | Medium name |
| `medium_url` | BacDive medium URL |
| `mediadive_url` | MediaDive medium URL |
| `isolation` | Isolation information |
| `isolation_source_categories` | Isolation source categories |
| `morphology_*` | Various morphology columns |
| `risk_assessment` | Risk assessment data |

## Data Flow

```
BacDive Transform
       |
       v
Resolves ncbitaxon_id (Case 1, 2, or 3 with fallback)
       |
       v
Writes to bacdive.tsv
       |
       v
MediaDive Transform reads bacdive.tsv
       |
       v
Uses pre-resolved ncbitaxon_id
```

## Implication

MediaDive automatically benefits from BacDive's fallback search - no changes needed to MediaDive.
