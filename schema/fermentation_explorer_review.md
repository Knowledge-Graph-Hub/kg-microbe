# Fermentation Explorer (clean DB) — schema & data review

**Source:** [thackmann/FermentationExplorer](https://github.com/thackmann/FermentationExplorer)
`database_clean.zip` → `data/raw/fermentation_explorer_database_clean.csv`
**Profile:** 20,806 records · 52 columns
**Schema:** [`schema/fermentation_explorer.yaml`](fermentation_explorer.yaml)
(generated by `scripts/generate_fermentation_explorer_schema.py`)
**Raw reference:** `data/raw/fermentation_explorer_database_raw.csv` (95-column per-source table)

## Verdict

The clean database is a well-structured, one-row-per-organism table consolidating
taxonomy, phenotype, and fermentation metabolism from six sources. It maps
cleanly onto a single `FermentationRecord` class with 13 controlled-vocabulary
enums; the schema and a 50-record sample validate. The data is **broad
taxonomically but sparse on its distinguishing metabolic annotations**, several
columns leak HTML presentation markup, and two lineage columns are entirely
placeholder — all manageable for KG integration.

---

## 1. Record identity & linkage

`LPSN ID` is unique and non-null (used as the schema identifier). Cross-reference
coverage to identifiers already used in KG-Microbe:

| External ID | Non-null | KG-Microbe node namespace |
| --- | ---: | --- |
| LPSN ID | 20,806 (100%) | — (record key) |
| BacDive ID | 18,963 (91%) | `bacdive` / `kgmicrobe.strain` |
| NCBI Taxonomy ID | 17,649 (85%) | `NCBITaxon` |
| GOLD Organism ID | 17,649 (85%) | — |
| GTDB ID | 14,094 (68%) | `GTDB` (genome accession) |
| IMG Genome ID | 13,967 (67%) | — |

No duplicate `(Genus, Species, Strain)` rows. **85–91% of records carry an
NCBITaxon, GTDB, or BacDive identifier**, so most can be joined directly to
existing KG-Microbe nodes rather than minting new ones.

## 3. Fermentation metabolism (the DB's distinguishing content)

| Field | Records | Coverage |
| --- | ---: | ---: |
| Type of metabolism = Fermentation | 3,270 | 16% |
| Type of metabolism = Methanogenesis | 131 | 1% |
| Type of metabolism = Other | 5,325 | 26% |
| *(metabolism unset)* | 12,080 | 58% |
| Fermentation substrates present | 2,475 | 12% |
| End products present | 1,595 | 8% |

The end-product / substrate annotations — the unique value of this resource for a
microbial KG — are present for a **minority** of records. End products,
major/minor end products, substrates, and fermentation substrates are
`;`-delimited chemical-name lists (ungrounded; candidates for CHEBI mapping).

## 4. Data-quality findings

| # | Severity | Finding |
| --- | --- | --- |
| 1 | **Medium** | `NCBI Taxonomy` and `Bergey Taxonomy` lineage columns are **100% placeholder** (`d__NA;p__NA;…`) — unpopulated. `GTDB Taxonomy` is placeholder on 6,712 rows (32%). Use `LPSN Taxonomy` (0% placeholder) / `GTDB Taxonomy` (where present) and the ID columns, not the NCBI/Bergey lineage strings. |
| 2 | Medium | 8 columns store **HTML anchor markup** (`<a href=…>…</a>`) rather than plain IDs/URLs: `LPSN Page link`, `GTDB ID link`, `GOLD Organism/Project ID link`, `NCBI Taxonomy ID link`, `IMG Genome ID link`, `BacDive ID link`, `Bergey Article link`. |
| 3 | Low | `Antibiotic sensitivity` mixes `;` and `,` delimiters on 12 rows (others are `;`); `GOLD Organism/Project ID` use `,` while most multi-value columns use `;`. |
| 4 | Info | Distinguishing metabolic annotations are sparse (end products 8%, fermentation substrates 12%, explicit fermentation/methanogenesis metabolism 16%). The table is taxonomically broad but metabolically partial. |

## 5. Schema conformance

- `schema/fermentation_explorer.yaml` validates with `gen-linkml`.
- A 50-record sample (`schema/examples/fermentation_explorer_sample.yaml`) passes
  `linkml-validate`.
- Enums are populated from the build's own distinct values, so every non-empty
  value of an enum column is covered by construction.

---

## 6. KG-Microbe integration notes & recommendations

1. **Join on existing identifiers.** Key records to KG nodes via `NCBI Taxonomy ID`
   (`NCBITaxon`), `GTDB ID` (`GTDB`), and `BacDive ID` (`bacdive`); mint a new
   `kgmicrobe` node only for the LPSN-only remainder.
2. **Ground the metabolic content.** Map end products / substrates / fermentation
   substrates to `CHEBI`, and `Type of metabolism` to `METPO` — this is the
   resource's main contribution to a fermentation KG.
3. **Reuse the isolation enums.** `Isolation category 1/2/3` are the same BacDive
   classification as [`bacdive_isolation_source_enums.yaml`](bacdive_isolation_source_enums.yaml)
   minus the `#` prefix; share/cross-reference rather than duplicating, and carry
   over the ENVO `meaning:` annotations.
4. **Clean the link columns** (extract plain URLs/IDs from the HTML) and **drop the
   all-placeholder NCBI/Bergey lineage columns** before transforming.
5. Map the organism phenotypes (gram stain, oxygen tolerance, cell shape,
   motility, spore formation) to `METPO` / `MICRO` / `OBA` for KG predicates.
6. Re-run `scripts/generate_fermentation_explorer_schema.py` when the upstream DB
   updates; enum drift surfaces in the diff.
